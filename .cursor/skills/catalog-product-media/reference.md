# Справка: расширенные правила (catalog-product-media)

## Два слоя

- **CSS (Next/Image):** как картинка ведёт себя в UI — основной способ в проекте.
- **Пакетная обработка:** `npm run catalog:normalize-images` — EXIF-поворот, даунскейл >1600px, пережатие JPEG; PNG/WebP без уменьшения размера не пересохраняются (иначе файл часто растёт). Для ~25k фото позже — отдельный WebP/CDN-пайплайн.

## Next.js

- В режиме `next start` компонент `Image` отдаёт оптимизированные форматы на лету — сырые PNG в `public/` могут оставаться как есть при умеренном размере.

## Техника файлов

- Экспорт в **sRGB**; при необходимости сброс чувствительных EXIF.
- **Прозрачный PNG:** на белой карточке проверять кайму; при артефактах — холст с белым фоном.

## Проверки

- Пути из `products.ts` существуют на диске.
- Один файл на два SKU — только осознанно.
- Заглушка `_pending.jpg` — единый стиль.

## SEO / фиды

- При появлении Merchant / маркетплейсов — отдельные требования к размеру и фону; не смешивать с вёрсткой магазина без явной задачи.

## Будущее

- 360°, видео: отдельное соглашение (постер, lazy, без автозвука).
