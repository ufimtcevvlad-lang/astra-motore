# Внутренний регламент обработки данных (чек-лист)

Этот документ используется менеджером и администратором сайта для ежедневной работы с персональными данными и юридическими запросами.

## 1) Что уже обязательно на сайте

- Обязательные согласия в формах: регистрация, вход, вход по СМС, корзина, запрос по идентификационному номеру автомобиля.
- Серверная валидация согласия на обработку персональных данных.
- Журнал согласий: `data/consent-events.ndjson` (дата/время, IP, user-agent, тип формы, версии документов).
- Баннер файлов cookies с выбором: только необходимые / все.
- Аналитика загружается только после согласия на аналитические cookies.

## 2) Контакт для запросов по персональным данным

Разместить и поддерживать отдельный контакт:

- адрес электронной почты для запросов субъектов данных;
- тема обращения: "Персональные данные";
- целевой срок первичного ответа: до 3 рабочих дней;
- срок закрытия обращения: до 30 календарных дней.

## 3) Обработка запросов пользователей

Типовые запросы:

- отзыв согласия на обработку персональных данных;
- удаление персональных данных;
- предоставление информации о составе и целях обработки;
- корректировка персональных данных.

Алгоритм:

1. Принять обращение и зафиксировать дату/канал.
2. Проверить личность заявителя (достаточный минимум для идентификации).
3. Передать ответственному сотруднику.
4. Выполнить действие (удаление, корректировка, выгрузка данных) и зафиксировать результат.
5. Отправить официальный ответ заявителю.

## 4) Сроки хранения данных (рекомендуемая матрица)

- заявки с сайта и запросы по идентификационному номеру автомобиля: до 3 лет;
- заказы и документы по сделке: не менее сроков, требуемых законодательством;
- учетные записи пользователей: пока активна учетная запись и до обоснованного срока после удаления;
- технические журналы безопасности: ограниченный срок, достаточный для расследования инцидентов.

## 5) Минимизация и безопасность

- не собирать избыточные поля в формах;
- ограничить доступ к журналам и базе по принципу минимальных прав;
- включить регулярные резервные копии;
- периодически проверять восстановление из резервной копии;
- фиксировать инциденты и меры реагирования.

## 6) Регулярный контроль

- ежемесячно: проверить актуальность ссылок и текстов юридических страниц;
- ежемесячно: проверить, что аналитика не загружается без согласия;
- ежеквартально: пересмотреть матрицу сроков хранения;
- при каждом изменении юридических текстов: обновить версии документов.
