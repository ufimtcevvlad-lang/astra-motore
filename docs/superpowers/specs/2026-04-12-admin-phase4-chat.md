# Фаза 4: Чат и заявки

## Контекст

Фазы 1–3 админ-панели Astra Motors завершены (авторизация, товары, категории, заказы). Фаза 4 — система чата: виджет на сайте для клиентов + единый инбокс в админке. Telegram-интеграция для клиентов — отдельная фаза позже.

**Существующая инфраструктура:**
- Таблицы БД: `conversations`, `messages`, `quick_replies` (созданы в фазе 1)
- Пункт сайдбара: "Чат и заявки" → `/admin/conversations`
- Telegram-бот для уведомлений админу (scripts/telegram-bot.mjs)

**Стек:** Next.js 16, React 19, SQLite (Drizzle ORM), PM2 + Nginx на VPS

---

## 1. Виджет чата на сайте

### Внешний вид
- Плавающая кнопка-пузырь в правом нижнем углу всех страниц магазина
- По клику — окно чата ~350x500px
- Бейдж с количеством непрочитанных на пузыре

### Начало чата
- При первом открытии — мини-форма: имя, телефон или email (обязательно одно из двух)
- После ввода — данные сохраняются в cookie (JWT-токен), при повторном визите чат открывается сразу
- История переписки сохраняется и видна клиенту

### Функции клиента
- Отправка текстовых сообщений
- Прикрепление файлов (фото, PDF, документы) — кнопка-скрепка + drag & drop
- Индикатор "Сообщение доставлено"
- Статус оператора: "Онлайн" / "Обычно отвечаем в течение 15 минут"

### Polling
- Каждые 3 секунды пока окно чата открыто
- Окно свёрнуто — polling останавливается

## 2. Инбокс в админке (`/admin/conversations`)

### Список диалогов (левая панель)
- Два раздела: **Непрочитанные** | **Все**
- Поиск по имени клиента / тексту сообщений
- Каждый элемент: имя клиента, канал (иконка: сайт/telegram), превью последнего сообщения, время, бейдж непрочитанных
- Сортировка: новые сообщения сверху
- Фильтр по назначенному админу

### Окно переписки (центральная панель)
- Лента сообщений: клиент слева, админ справа
- Внутренние заметки — жёлтый фон, видны только админам
- Поле ввода с кнопкой прикрепления файлов
- Кнопка быстрых ответов — выпадающий список шаблонов, текст вставляется в поле (не отправляется)

### Информация о клиенте (правая колонка)
- Имя, телефон, email
- Канал обращения
- Назначенный админ (выпадающий список для смены)
- Кнопка "Закрыть диалог" (архивация)

### Polling
- Каждые 3 секунды на странице чата
- Каждые 30 секунд — обновление счётчика непрочитанных в сайдбаре

## 3. Быстрые ответы (шаблоны)

Управление в настройках админки (отдельная страница `/admin/settings/quick-replies`):
- Создание: заголовок + текст шаблона
- Редактирование, удаление
- Сортировка (drag & drop или стрелки)

## 4. Уведомления

### В админке
- Счётчик непрочитанных в сайдбаре рядом с "Чат и заявки"
- Автообновление через polling

### Браузерные пуши (Web Push)
- При первом входе в админку — запрос разрешения
- При новом сообщении от клиента — push с именем клиента и превью текста
- Клик по уведомлению — переход в диалог

### Telegram
- Существующий бот шлёт уведомление: "Новое сообщение от [Имя]: [превью]"
- Без возможности ответить через Telegram (будет в отдельной фазе)

### Звук
- Короткий звук при новом сообщении (вкладка админки открыта)
- Настройка вкл/выкл в настройках админки

## 5. API маршруты

### Клиентский виджет

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/chat/start` | Начать диалог (имя, контакт) → conversationId + JWT token |
| GET | `/api/chat/messages?conversationId=X&after=Y` | Новые сообщения (polling) |
| POST | `/api/chat/send` | Отправить сообщение |
| POST | `/api/chat/upload` | Загрузить файл |

Авторизация: JWT-токен в cookie, выдаётся при старте диалога.

### Админка

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/admin/conversations` | Список диалогов (фильтр: unread/all, поиск, пагинация) |
| GET | `/api/admin/conversations/[id]` | Диалог + сообщения |
| POST | `/api/admin/conversations/[id]/send` | Отправить сообщение / заметку |
| PUT | `/api/admin/conversations/[id]/assign` | Назначить админа |
| PUT | `/api/admin/conversations/[id]/read` | Пометить прочитанным |
| GET | `/api/admin/conversations/unread-count` | Счётчик для сайдбара |
| POST | `/api/admin/conversations/upload` | Загрузить файл |

### Быстрые ответы

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/admin/quick-replies` | Список шаблонов |
| POST | `/api/admin/quick-replies` | Создать |
| PUT | `/api/admin/quick-replies/[id]` | Редактировать |
| DELETE | `/api/admin/quick-replies/[id]` | Удалить |

## 6. Техническая реализация

### Изменения в БД
- `conversations`: добавить `assigned_admin_id` (FK → admins), `last_message_at`, `unread_count`
- `messages`: добавить `read_at`
- Новая таблица `chat_tokens`: `id`, `conversation_id`, `token`, `expires_at`
- Новая таблица `push_subscriptions`: `id`, `admin_id`, `endpoint`, `keys_json`, `created_at`

### Виджет чата
- React-компонент в корневом layout сайта (`src/app/layout.tsx`)
- Состояние свёрнут/развёрнут — localStorage
- Cookie `chat_token` — идентификация клиента при повторном визите

### Загрузка файлов
- Макс. размер: 10MB
- Разрешённые типы: jpg, png, webp, gif, pdf, doc, docx, xls, xlsx
- Сохранение: `public/uploads/chat/[conversationId]/[uuid].[ext]`
- Оригинальное имя — в JSON-поле `attachments` сообщения

### Web Push
- Service Worker для приёма пушей
- VAPID-ключи генерируются при первом деплое
- Подписки хранятся в `push_subscriptions`
- Отправка через npm-пакет `web-push`

### Telegram-уведомления
- HTTP-запрос к Telegram Bot API при новом сообщении от клиента
- Используем `TELEGRAM_BOT_TOKEN` + `TELEGRAM_ADMIN_CHAT_ID` из env
