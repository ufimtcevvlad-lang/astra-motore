# Идеальная карточка товара — GM Shop 66

**Дата**: 2026-04-13
**Статус**: Утверждено
**Цель**: Переработать карточку товара так, чтобы она была идеальной по дизайну, SEO, конверсии и масштабируемости на 7000+ товаров. Больше не возвращаемся ни к дизайну, ни к структуре.

---

## 1. SEO-разметка

### 1.1 Title

**Формула**: `{sku} — {name} | GM Shop 66`

Примеры:
- `W 712/95 — Фильтр масляный MANN | GM Shop 66`
- `25403 01 — Опора двигателя LEMFOERDER | GM Shop 66`

`name` уже содержит бренд (из 1С / после чистки). Не дублируем.

Артикул первым — это главный поисковой запрос. Бренд магазина в конце для узнаваемости.

### 1.2 Meta Description

**Шаблон с вариациями** (6 финальных фраз, рандомный выбор при генерации):

С полными данными:
> `{brand} {sku} — {name} для {car}. Арт. {sku}, в наличии в Екатеринбурге. Доставка по РФ. Гарантия. GM Shop 66.`

Без car:
> `{brand} {sku} — {name}. Арт. {sku}, в наличии в Екатеринбурге. Доставка по РФ. Гарантия. GM Shop 66.`

Без brand:
> `{sku} — {name}. Арт. {sku}, в наличии в Екатеринбурге. Доставка по РФ. GM Shop 66.`

Максимум 158 символов. Обрезка по слову если превышает.

### 1.3 Meta Keywords (для Яндекса)

Формула: `[sku, brand, название детали (без бренда), марка авто, "автозапчасти Екатеринбург"]`

Пример: `["W 712/95", "MANN", "фильтр масляный", "Opel Astra J", "автозапчасти Екатеринбург"]`

Только заполненные поля попадают в массив. Без мусорных слов.

### 1.4 Open Graph

```
og:type        → "product"
og:title       → то же что title
og:description → то же что meta description
og:url         → canonical URL
og:image       → первое фото товара или fallback brand logo
og:locale      → "ru_RU"
og:site_name   → "GM Shop 66"
product:price:amount   → цена
product:price:currency → "RUB"
```

Twitter Card не добавляем (неактуален для РФ).

## 3. Сайдбар (правая колонка)

Sticky на десктопе (`md:sticky md:top-24`). Белый фон, скруглённый, тень, янтарная полоска сверху.

Содержимое сверху вниз:

### 3.1 Цена + Избранное
- Цена: крупно, `text-3xl font-bold`
- Сердечко: кнопка справа, без изменений

### 3.2 Наличие
- **> 5 шт.**: `✓ В наличии` — мелкий зелёный текст, без количества
- **3–5 шт.**: `Осталось N шт.` — янтарный цвет, привлекает внимание
- **0 шт.**: `Нет в наличии` — серый текст + кнопка `Спросить о сроках` → WhatsApp

### 3.3 Количество + «В корзину»
Без изменений. Текущая реализация работает хорошо.

### 3.4 Инфо-карточка (переехала сюда)
Компактная `<dl>` с разделителями:
- **Артикул** — `font-mono font-semibold` + кнопка копирования
- **Бренд** — если заполнен
- **Авто** — если заполнен

Поле «Страна» убираем — бесполезно для покупателя.
Пустые поля не рендерятся.

### 3.5 Быстрая связь
Кнопка: `Спросить про эту деталь`
Ведёт в WhatsApp с предзаполненным сообщением:
`Здравствуйте! Интересует {name}, арт. {sku}. Есть в наличии?`

Стиль: текстовая ссылка с иконкой WhatsApp, не кнопка. Ненавязчиво но заметно.

### 3.6 Доставка / Гарантия / Возврат
Три ссылки с иконками. Текущая реализация — без изменений.

- ~~Дисклеймер «менеджер свяжется»~~

## 4. Галерея изображений

### 4.1 С фотографиями
Текущая реализация сохраняется:
- Основное фото + миниатюры
- Стрелки навигации (hover на десктопе)
- Лайтбокс (полноэкранный просмотр по клику)
- Свайп на мобильных
- Счётчик `1/N`
- Кнопка «Поделиться» в правом верхнем углу

**Добавляем**: зум при наведении на основное фото (десктоп). При hover — увеличенный фрагмент в области курсора. Важно для маркировок и артикулов на упаковке.

### 4.2 Без фотографий (плейсхолдер)
- Фон: `#0b1220` (brand dark)
- Иконка: стилизованная коробка/деталь, `text-slate-600`
- Текст: `Фото скоро появится`, `text-slate-500`
- Соотношение сторон: то же что у обычных фото
- Не выглядит как ошибка — выглядит как ожидание

## 5. Описание товара (автогенерация)

### 5.1 Короткое описание (шаблонное, всегда есть)
Генерируется автоматически при создании/сохранении товара. Всегда видно на странице.

**Формула**:
```
{name} ({brand}) — {тип_фраза} для {car}. Артикул {sku}. {финальная_фраза}.
```

Адаптация при пустых полях:
- Без brand: `{name} — {тип_фраза}. Артикул {sku}. {финальная_фраза}.`
- Без car: убираем `для {car}`
- Без brand и car: `{name}. Артикул {sku}. {финальная_фраза}.`

**Тип_фраза** (выбирается детерминированно (hash от ID товара) из вариантов):
1. `расходник`
2. `деталь`
3. `запчасть`
4. `автокомпонент`
5. `комплектующая`

Используется только если нет более точной категории. Если категория = «Фильтры», тип_фраза = «фильтрующий элемент». Маппинг категория → фраза в конфиге.

**Финальная_фраза** (детерминированно (hash от ID товара) из 6 вариантов):
1. `В наличии, отправка из Екатеринбурга.`
2. `Склад Екатеринбург, доставка по РФ.`
3. `Есть на складе, быстрая отправка.`
4. `В наличии в Екатеринбурге, отправим по РФ.`
5. `На складе, доставка по России.`
6. `Готов к отправке из Екатеринбурга.`

Вариативность — чтобы Яндекс не пометил как дублированный контент.

### 5.2 Длинное описание (опциональное)
Если поле `longDescription` заполнено — показывается под коротким описанием в сворачиваемом блоке «Подробнее».

Для большинства товаров его не будет. Это нормально.

### 5.3 Перегенерация
При сохранении товара в админке — короткое описание перегенерируется автоматически из текущих полей. Добавил бренд → описание обновилось. Добавил авто → описание обновилось.

Ручное описание (если заполнено вручную) имеет приоритет — шаблон не перезаписывает ручной текст.

## 7. Похожие товары

Одна секция вместо двух («рекомендуем» + «аналоги дешевле»).

**Заголовок**: `Похожие товары`

**Содержимое**: товары из той же категории (исключая текущий). Горизонтальный скролл карточек `CatalogProductCard`.

**Максимум**: 8 карточек.

**Условие рендеринга**: минимум 2 похожих товара. Иначе секция не показывается.

**Убираем**: слово «аналог», бейджи «аналог», сравнение цен. Без данных о совместимости называть товары аналогами некорректно.

## 9. Поведение при неполных данных

| Поле | Пустое → Поведение |
|------|---------------------|
| Фото | Плейсхолдер (тёмный фон, иконка, текст) |
| Бренд | Не показывается в инфо-карточке, убирается из шаблонов |
| Авто (car) | Не показывается в инфо-карточке, убирается из шаблонов |
| Описание | Автогенерация короткого описания — поле никогда не пустое |
| longDescription | Блок «Подробнее» не рендерится |
| Спецификации | Секция «Характеристики» не рендерится |
| Похожие товары | Секция не рендерится |

**Минимальная карточка** (только из 1С: название, артикул, цена):
- H1: `{sku} — {name}`
- Галерея: плейсхолдер
- Сайдбар: цена, наличие, «В корзину», WhatsApp, доставка/гарантия/возврат
- Инфо: только артикул
- Описание: шаблонный текст без бренда и авто
- Никаких пустых секций или «нет данных»

## 11. Не в скоупе

- Мобильная оптимизация (плавающая кнопка, миниатюры) — откладываем
- Технические характеристики и совместимость с авто — платные данные, не делаем
- LLM-генерация описаний — слишком дорого, рискованно, не масштабируется
- Яндекс.Метрика e-commerce dataLayer — отдельная задача
