# Фаза 2: Товары и Категории — Спецификация

## Контекст

Админ-панель Astra Motors. Фаза 1 (БД, авторизация, каркас) завершена и замержена в main.
В базе: 121 товар, 13 категорий, 7 групп, 31 бренд. На складе 7000+ позиций.

## Раздел «Товары» (`/admin/products`)

### Список товаров

- **Формат**: карточки-строки — фото 56px, название + (артикул, бренд, категория) под ним, цена, статус наличия, кнопка редактирования
- **Пагинация**: 20 товаров на странице
- **Фильтры** (панель над таблицей):
  - Поиск по названию или артикулу (текстовое поле)
  - Категория (dropdown)
  - Бренд (dropdown)
  - Наличие (все / в наличии / нет в наличии)
  - Диапазон цены (от — до)
- Кнопка «+ Добавить товар» — переход на `/admin/products/new`
- Клик по строке или иконке ✎ — переход на `/admin/products/[id]`

### Форма товара (`/admin/products/new`, `/admin/products/[id]`)

Секции формы:

**Основные данные:**
- Название (`name`, text, обязательное)
- Артикул (`sku`, text, обязательное)
- Бренд (`brand`, text, обязательное)
- Автомобиль (`car`, text)
- Цена (`price`, number, обязательное, в рублях)
- Наличие (`inStock`, number, 0 = нет в наличии)

**Медиа:**
- Главное фото (`image`) — загрузка файла, превью
- Дополнительные фото (`images`) — множественная загрузка, drag-and-drop сортировка

**Тексты:**
- Описание (`description`, textarea)
- Длинное описание (`longDescription`, textarea, опциональное)

**Связи:**
- Категория (`categoryId`) — dropdown из существующих категорий
- Характеристики (`productSpecs`) — динамический список пар «Название — Значение», кнопки добавить/удалить
- Аналоги (`productAnalogs`) — поле поиска по названию/артикулу, выбор из результатов, список привязанных аналогов с кнопкой удалить

**Действия:**
- Кнопка «Сохранить» (создание или обновление)
- Кнопка «Удалить» (только при редактировании, с модальным подтверждением)
- При удалении: каскадно удаляются specs и analogs (настроено в схеме БД)

### Импорт из Excel (`/admin/products/import`)

**Шаг 1 — Загрузка файла:**
- Поле загрузки `.xlsx` файла
- Формат файла: колонка C = наименование, D = артикул, E = бренд, F = цена (формат `artikuly.xlsx` из 1С)

**Шаг 2 — Превью и разрешение конфликтов:**
- Парсинг файла на сервере
- Показать две таблицы:
  - **Новые товары** — товары, артикул которых не найден в БД. Показать название, артикул, бренд, цена.
  - **Совпадения (дубли)** — товары, артикул которых уже есть в БД. Для каждого показать текущие значения vs новые значения. Чекбокс «Обновить» / «Пропустить» для каждого (по умолчанию — пропустить).
- Итого: «Будет добавлено: N, будет обновлено: M, пропущено: K»

**Шаг 3 — Импорт:**
- Кнопка «Импортировать»
- Прогресс-бар (для больших файлов)
- Результат: «Добавлено: N, обновлено: M, пропущено: K, ошибок: E»

## Раздел «Категории» (`/admin/categories`)

### Список категорий

- Группировка по `group_name` — заголовок группы, под ним категории этой группы
- Для каждой категории: название, slug, количество привязанных товаров
- Drag-and-drop сортировка внутри группы (обновление `sortOrder`)
- Кнопка «+ Добавить категорию»

### Форма категории (`/admin/categories/new`, `/admin/categories/[id]`)

- Название (`title`, text, обязательное)
- Slug (`slug`, text, обязательное, авто-генерация из названия через транслитерацию)
- Группа (`groupName`, text с автодополнением из существующих групп)
- Slug группы (`groupSlug`, авто-генерация из groupName)
- Порядок сортировки (`sortOrder`, number)

**Действия:**
- Сохранить
- Удалить — только если к категории не привязаны товары. Если привязаны — показать предупреждение с количеством товаров.

## API роуты

### Товары

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/admin/products` | Список с фильтрацией и пагинацией |
| POST | `/api/admin/products` | Создание товара |
| GET | `/api/admin/products/[id]` | Получение товара со specs и analogs |
| PUT | `/api/admin/products/[id]` | Обновление товара |
| DELETE | `/api/admin/products/[id]` | Удаление товара |
| POST | `/api/admin/products/import` | Загрузка и парсинг Excel, возврат превью |
| POST | `/api/admin/products/import/confirm` | Подтверждение импорта с выбранными действиями |

### Категории

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/admin/categories` | Список всех категорий |
| POST | `/api/admin/categories` | Создание категории |
| GET | `/api/admin/categories/[id]` | Получение категории |
| PUT | `/api/admin/categories/[id]` | Обновление категории |

## Технический подход

- **ORM**: Drizzle ORM (уже в проекте), все запросы к SQLite через существующий `db` из `src/app/lib/db/index.ts`
- **Схема БД**: таблицы `products`, `productSpecs`, `productAnalogs`, `categories` уже существуют, изменения схемы не требуются
- **Excel-парсинг**: библиотека `xlsx` (SheetJS) — парсинг на сервере в API роуте
- **Загрузка фото**: сохранение в `public/uploads/products/`, генерация уникальных имён файлов
- **Компоненты**: серверные компоненты для списков (SSR), клиентские для форм и интерактивных элементов
- **Валидация**: на сервере в API роутах, ошибки возвращаются как JSON `{ error: string, field?: string }`
- **Стилизация**: Tailwind CSS 4 (уже в проекте), тёмная тема админки (фон `#1e1e2e` / `#16161e`)
